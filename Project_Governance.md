# Project Governance

## Document Responsibilities

### WHAT to build

1. PRD.md
2. FSD.md
3. TDD.md
4. DESIGN.md

### WHEN to build

PHASE_ROADMAP_AUDIT.md

### HOW to build

IMPLEMENTATION.md

## Phase Rule

A feature is considered part of a phase only if
PHASE_ROADMAP_AUDIT.md assigns it to that phase.

Product requirements appearing in PRD/FSD/TDD do not
override roadmap sequencing.

## Early Implementation Rule

Code implemented ahead of its roadmap phase must be marked:

Early Implementation

and does not affect completion of the current phase.
